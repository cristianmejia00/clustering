# Bibliometric Clustering Pipeline

Analysis pipeline for clustering academic articles, news, and patents using citation networks and topic models.

## Quick Start

### 1. Configure Paths in YAML

Set your bibliometrics root path directly in the configuration files.

### 2. Configure Pipeline

Edit the two YAML config files in the project root:

- **`config_dataset.yml`** — Data source paths, file filtering, column selection, embedding parameters, and network type.
- **`config_analysis.yml`** — Analysis parameters: clustering algorithm, thresholding, topic model options, LLM prompts, and report settings.

### 3. Run By Task (Recommended)

Run from an R session with working directory at repo root.

Unified launcher:

- `source("scripts/run_pipeline.R")`
- `run_pipeline(c("dataset", "analysis", "reports", "ai", "charts"))` — full pipeline
- `run_pipeline(c("reports", "ai", "charts"))` — rerun from reports
- `run_pipeline(c("ai", "charts"))` — AI enrichment + charts only
- `run_pipeline(c("charts"))` — charts only

Valid stages: `dataset`, `analysis`, `reports`, `ai`, `charts`. Supply any subset in any order.

1. Dataset routine (mandatory embeddings included):
   Run `source("scripts/dataset_only.R")`.
   Produces cleaned dataset, citation network, and embeddings for all embed profiles.
2. Analysis routine (clustering only):
   Run `source("scripts/analysis_only.R")`.
3. Reports routine (report tables):
   Run `source("scripts/reports_only.R")`.
4. AI enrichment:
   Run `source("scripts/ai_only.R")`.
5. Charts:
   Run `source("scripts/charts_only.R")`.

Embeddings are now part of the dataset pipeline for all datasets because they are also used later for optional cosine-similarity analyses, not only for topic modeling.

### 4. Dataset Artifacts (Now Required)

For each embed profile in `config_dataset.yml` (e.g., `e01`), the dataset routine writes:

- `corpus.csv`
- `embeddings.npy`
- `embeddings_ids.json`
- `embeds_settings.json`

These are saved under:

- `<bibliometrics_directory>/<project_folder>/<from_filtered_dataset>/<embed_profile>/`

### 5. Topic Model Flow

Topic-model notebooks should consume precomputed embeddings generated by `scripts/dataset_only.R`.

### 6. System Paths

`utils/system_paths.R` resolves the cloud-storage root (`output_folder_path`) per OS (macOS, Windows, Linux). Update paths there if your mount point differs.

### 7. Legacy Entry Scripts

The original root scripts (`a00` to `a05`) still work and are kept for compatibility, but new users should use the scripts under `scripts/`.

Canonical dataset scripts now live at:

- `pipelines/dataset/loader.R`
- `pipelines/dataset/network.R`

Canonical analysis scripts now live at:

- `pipelines/analysis/citation_network/components.R`
- `pipelines/analysis/citation_network/clustering.R`
- `pipelines/analysis/citation_network/thresholding.R`

Canonical shared reports entrypoint:

- `pipelines/reports/generator.R`

## Project Structure

```text
config_dataset.yml          # Dataset configuration (YAML)
config_analysis.yml         # Analysis configuration (YAML)
a00–a05_*.R                 # Main pipeline scripts (run in order)
scripts/                    # Recommended entry points (dataset/analysis/full)
pipelines/dataset/          # Canonical dataset scripts (loader, network)
pipelines/analysis/citation_network/ # Canonical citation-network analysis scripts
pipelines/reports/          # Canonical shared reports entrypoint
pipelines/dataset/utils/01_data_loading/     # Migrated data-loading helpers
pipelines/dataset/utils/02_citation_network/ # Migrated network clustering helpers
03_reports/                 # Report generators
04_charts/                  # Visualization scripts
05_llm/                     # LLM integration (Claude API)
06_quarto/                  # Quarto document generation
yy_heatmap_sankey/          # Heatmap and Sankey diagrams
utils/                   # Shared utilities and config loader
assets/                  # Static assets (colors, overlays, credentials)
```
